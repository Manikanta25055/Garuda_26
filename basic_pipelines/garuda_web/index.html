Documentation

Garuda Security System

AI-powered home security running on Raspberry Pi 5 with Hailo-8L neural processor. Real-time object detection at 30+ FPS with configurable danger alerts.

Detection Pipeline

1 Camera capture
→
2 Hailo inference
→
3 Post-processing
→
4 Alert logic

System Modes

Control system behavior with toggleable modes: Privacy Blur obscures faces, Night Mode enables high-priority alerts, DND suppresses all alerts, Emergency overrides DND for critical situations.

Detection System

How it works

YOLOv6n model runs on the Hailo-8L at 30+ FPS. Each frame is analyzed for objects matching COCO class labels. Detections above the confidence threshold are logged.

Alert Logic

1 Object detected
→
2 Matches danger label?
→
3 Trigger alert
→
4 Send email

Categories

Danger labels (default: scissors) trigger visual + audio + email alerts. Watch labels (e.g. person, backpack) are silently logged without triggering alerts.

Detection threshold (0.05–0.95) controls sensitivity. Lower values detect more objects but may produce false positives.

Authentication

Login Flow

1 Credentials
→
2 OTP via email
→
3 Session token

Users log in with username + password. Admins require an additional OTP sent via email. Master Keys provide instant admin access and unlock the logs page.

Session Management

Sessions expire after 1 hour (admin) or 5 days (user with "Remember me"). Tokens are stored as HTTP-only cookies for same-origin requests and in localStorage for cross-origin access.
